# Phase 1：MCP 浏览器 Example

> 用途：冻结新的 `Phase 1` 标准验收场景，证明最小可运行 agent 能通过 MCP 工具完成浏览器操作、信息提取与结果分析。

## 1. Example 目标

使用 `chrome-devtools` MCP 作为标准工具场景，验证 agent 能：

- 通过提示词触发浏览器操作
- 打开指定公开页面
- 读取页面标题与主要内容
- 基于读取到的页面信息进行分析并输出结果

## 2. 固定工具与任务类型

- 工具：`chrome-devtools` MCP
- 任务类型：浏览器打开页面、读取内容、分析并输出

## 3. 标准提示词样例

```text
请使用 chrome-devtools 工具打开指定公开网页，读取页面标题和主要内容，提取关键要点，并输出一份中文分析结果。

要求：
1. 必须通过浏览器工具访问页面，而不是凭空假设内容。
2. 先给出页面标题。
3. 再总结页面的核心内容和 3-5 个关键要点。
4. 最后给出简短分析结论。
```

## 4. 验收关注点

- agent 是否真的调用了浏览器工具
- 是否真正读取了页面内容而不是臆测
- 是否把浏览器获取的信息转化成分析输出
- 是否保持“工具获取事实，模型负责归纳”的行为链条

## 5. 不校验的内容

- 不要求页面内容逐字精确匹配
- 不要求固定某一个公开页面地址
- 不要求在本仓库提供真实可执行脚本

## 6. 最小完成产物

- 一条固定的 `chrome-devtools` MCP 验收场景
- 一份可直接复用的提示词样例
- 一组能判断“最小可运行 agent 已具备真实工具使用能力”的验收关注点
