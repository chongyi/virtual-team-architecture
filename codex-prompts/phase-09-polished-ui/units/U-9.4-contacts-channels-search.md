# U-9.4 联系人、频道与搜索

## 目标 (Goal)

重塑通讯录、频道列表和全局搜索界面：联系人列表（含在线状态和部门分组）、频道列表（未读/置顶/分类）、全局搜索（消息/联系人/文件）。

## 上下文 (Context)

- 前置：U-9.3（聊天体验完成）
- 当前通讯录用静态 mock 数据，频道列表简陋

## 工作范围

### 1. 通讯录 — `contacts_screen.dart`

**顶部**：搜索框（`TextField`，圆角 8px，左侧搜索图标，背景 `bgSecondary`）

**分段切换**：水平 `TabBar`，3 个标签：

| 标签 | 内容 |
|------|------|
| 全部 | 所有联系人和 VE |
| 同事 | 按组织/部门分组的用户列表 |
| 虚拟员工 | 仅 VE 列表，每个含能力标签 |

**联系人卡片**（`ContactTile`）：
- 左侧：`AppAvatar` + 在线状态圆点
- 中间：名称（`title2`）+ 部门/头衔（`caption1`，`textTertiary`）
- VE 联系人：名称后有 "AI" 标签（药丸形，`colorPrimaryLight` 背景）
- 右侧：可选的快捷操作图标（发起聊天）
- 点击进入私信频道

**分组标题**（`headerBuilder`）：组织/部门名称，左对齐，`overline` 字号，`textTertiary`，含 8px 上下内边距。

**索引条**（可选）：右侧快速定位字母条（类似通讯录 A-Z）

### 2. 频道列表 — `channel_list_screen.dart`

**搜索 + 筛选**：顶栏搜索框 + 频道类型过滤（all/direct/group/channel）。

**频道卡片**：
- 左侧：频道图标 + 类型标签
- 中间：频道名称 + 最后一条消息预览
- 右侧：未读角标（数字）+ 最后消息时间

**排序**：按最后消息时间降序，置顶频道在最上。

**置顶频道**：背景 `primaryLight` 10% 透明度，左侧 3px 主色条。

**频道操作**（长按或右滑）：置顶/取消置顶、标记已读、退出频道（确认弹窗）。

### 3. 全局搜索 — `search_screen.dart`

**入口**：点击顶栏搜索图标，全屏覆盖搜索页面。

**搜索框**：顶部大搜索框（高度 48px），自动聚焦，输入时实时搜索（300ms 防抖）。

**结果分区**（水平 TabBar）：

| 标签 | 搜索范围 | 高亮 |
|------|---------|------|
| 消息 | 全文搜索（FTS） | 关键词用 `<mark>` 标签，MarkdownBody 渲染 |
| 联系人 | 名称匹配 | 匹配文字高亮主色 |
| 文件 | 文件名匹配 | 文件名高亮 |

**结果卡片**（消息搜索结果）：
- 发送者头像 + 名称
- 匹配的消息片段（含高亮）+ 来源频道名
- 点击跳转到该消息所在的频道

**空状态**：搜索图标 + "未找到相关结果" + "尝试其他关键词"。

## 约束 (Constraints)

- 全部使用 U-9.1 设计 Token
- 通讯录数据从真实 API 获取（替换 mock），VE 联系人保持现有 `VeContactService`
- 搜索防抖 300ms，避免频繁请求
