# U-9.8 深色模式与无障碍访问

## 目标 (Goal)

实现完整的深色主题（Dark Mode），并对全应用进行无障碍（Accessibility）适配：语义标签、足够的对比度、可缩放文字、屏幕阅读器（VoiceOver/TalkBack）支持。

## 上下文 (Context)

- 前置：U-9.7（状态和动效完成）
- 当前仅有浅色主题，无深色模式

## 工作范围

### 1. 深色模式设计 Token

在 `design_tokens.dart` 中添加完整的深色 Token。**核心原则：深色模式下不是简单反转颜色，而是降低对比度、减少纯黑、用深灰替代**。

```dart
// 深色模式 Token（示例）
static const darkBgPrimary = Color(0xFF1E1E1E);     // 页面背景（非纯黑）
static const darkBgSecondary = Color(0xFF252525);   // 卡片/输入框/气泡
static const darkBgTertiary = Color(0xFF2D2D2D);    // 悬浮态
static const darkSurface = Color(0xFF333333);       // 弹窗/面板

static const darkTextPrimary = Color(0xFFE8EAED);
static const darkTextSecondary = Color(0xFF9AA0A6);
static const darkTextTertiary = Color(0xFF80868B);

static const darkBorderLight = Color(0xFF3C4043);
static const darkBorderDefault = Color(0xFF5F6368);
```

深色模式下的语义色调整：
- `colorPrimary`：保持主色，但 `primaryLight` 背景用 20% 透明度
- 消息气泡：自己的气泡用主色 80% 透明度背景，他人的气泡用 `darkBgSecondary`
- 在线状态色保持原有语义色（绿色仍为绿色），但降低饱和度

### 2. `AppTheme.dark()` 方法

完整的 `ThemeData.dark()` 配置，使用 Material 3 的 `ColorScheme.dark()` 作为基础：

```dart
static ThemeData dark() {
  return ThemeData(
    useMaterial3: true,
    brightness: Brightness.dark,
    colorScheme: ColorScheme.dark(
      primary: AppTokens.colorPrimary,
      onPrimary: AppTokens.textOnPrimary,
      surface: AppTokens.darkBgPrimary,
      onSurface: AppTokens.darkTextPrimary,
      // ... 完整映射
    ),
    scaffoldBackgroundColor: AppTokens.darkBgPrimary,
    // ... 同上 light() 的完整覆盖
  );
}
```

所有 Flutter 组件主题覆盖必须与 `light()` 完全对应（AppBar/Card/Input/BottomNav/NavBar/Divider/Chip/Dialog/SnackBar/TabBar/ListTile/FAB）。

### 3. 主题切换机制

在 `lib/app/app.dart` 中：
- 使用 `ValueNotifier<ThemeMode>` 或 `Riverpod StateProvider<ThemeMode>` 持有当前主题模式
- 监听系统深色模式变化（`PlatformDispatcher.instance.onPlatformBrightnessChanged`）
- 当用户选择"跟随系统"时自动切换
- 主题变更通过 `ValueListenableBuilder` 或 `Consumer` 重建 `MaterialApp`

在 U-9.5 的个人设置页面中接入切换（已预留 UI），三种模式：
- 浅色（`ThemeMode.light`）
- 深色（`ThemeMode.dark`）
- 跟随系统（`ThemeMode.system`）

主题偏好持久化到 `SharedPreferences` 或 Drift 本地数据库。

### 4. 无障碍适配

**语义标签**（Semantics Label）：
- 所有图标按钮添加 `Semantics(label: "搜索")` / `tooltip: "搜索"`
- 头像添加 `Semantics(label: "张三的头像")`
- 在线状态添加 `Semantics(label: "在线")`
- 未读角标添加 `Semantics(label: "3条未读消息")`
- VE 联系人添加 `Semantics(label: "虚拟员工：知识库助理，在线")`

**对比度检查**：
- 确保浅色模式下文字与背景对比度 >= 4.5:1（WCAG AA 标准）
- 确保深色模式下文字与背景对比度 >= 4.5:1
- 检查主色按钮上的白色文字对比度
- 检查 `textTertiary` 与背景在两种模式下都能通过 AA 标准（如不通过，调整为 `textSecondary`）

**可缩放文字**：
- 所有文字使用 `MediaQuery.of(context).textScaler` 自动适配系统字体大小设置
- 不使用 `textScaleFactor: 1.0` 强制禁用缩放（仅在图标字体等场景禁用）

**触控目标**：
- 所有可点击元素的最小触控区域为 48x48px（Material 标准）
- 检查所有 `IconButton`、`InkWell`、`GestureDetector` 应用的间距是否足够

**焦点导航**（桌面端 Tab 键导航）：
- 为所有交互元素添加 `FocusNode` 和 `FocusTraversalOrder`
- 确保 Tab 键能按逻辑顺序遍历所有可操作控件

### 5. 动态色彩（Android 12+ Material You）

在 Android 12+ 上可选启用动态色彩：
```dart
if (Platform.isAndroid) {
  return DynamicColorBuilder(
    builder: (lightColorScheme, darkColorScheme) {
      return MaterialApp(/* ... */);
    },
  );
}
```
动态色彩作为可选项，在设置中通过开关控制。
