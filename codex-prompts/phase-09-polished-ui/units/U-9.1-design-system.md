# U-9.1 设计系统基础

## 目标 (Goal)

建立完整的设计 Token 体系（Design System Foundation）：全局色彩方案、字体层级、间距规范、圆角体系、阴影层级、图标集，使得后续所有 UI 单元可以直接引用这些 Token 而非硬编码数值。

> 这是最重要的奠基单元。所有后续 UI 单元都依赖本单元的设计 Token。Codex 在执行后续单元时必须严格引用本单元定义的 Token，不得自创颜色或尺寸。

## 上下文 (Context)

- 前置：Phase 6 全部完成
- 设计参考：企业微信（WeChat Work）、飞书（Feishu）、Slack、Linear、Notion、Discord
- 当前 Flutter 应用位于 `apps/flutter/`，使用 Material 3 基础主题

## 设计方向

**整体风格**：专业、克制、高效的企业协作风格，避免花哨，强调信息层级和操作效率。

## 工作范围

### 1. 设计 Token 文件

创建 `apps/flutter/lib/design/` 目录，包含以下文件：

#### `design_tokens.dart` — 全局 Token 定义

```dart
// 所有设计常量集中定义，禁止在其他文件中硬编码颜色/字号/间距
class AppTokens {
  // 色彩系统
  static const colorPrimary = Color(0xFF1A73E8);      // 主色 — Google Blue 风格
  static const colorPrimaryDark = Color(0xFF1557B0);
  static const colorPrimaryLight = Color(0xFFE8F0FE);
  static const colorSuccess = Color(0xFF0F9D58);       // 成功/在线
  static const colorWarning = Color(0xFFF4B400);       // 警告
  static const colorDanger = Color(0xFFDB4437);        // 危险/删除
  static const colorInfo = Color(0xFF4285F4);          // 信息
  
  // 中性色（按亮度 0-1000 分 10 级）
  static const gray0 = Color(0xFFFFFFFF);    // 白
  static const gray50 = Color(0xFFF8F9FA);
  static const gray100 = Color(0xFFF1F3F4);
  static const gray200 = Color(0xFFE8EAED);
  static const gray300 = Color(0xFFDADCE0);
  static const gray400 = Color(0xFFBDC1C6);
  static const gray500 = Color(0xFF9AA0A6);
  static const gray600 = Color(0xFF80868B);
  static const gray700 = Color(0xFF5F6368);
  static const gray800 = Color(0xFF3C4043);
  static const gray900 = Color(0xFF202124);
  static const gray1000 = Color(0xFF000000); // 黑
  
  // 语义背景色
  static const bgPrimary = gray0;
  static const bgSecondary = gray50;
  static const bgTertiary = gray100;
  static const bgHover = Color(0x0A000000);  // 10% 黑叠加
  
  // 文字色
  static const textPrimary = gray900;
  static const textSecondary = gray700;
  static const textTertiary = gray500;
  static const textDisabled = gray400;
  static const textOnPrimary = gray0;
  
  // 边框色
  static const borderLight = gray200;
  static const borderDefault = gray300;
  
  // ... 其余
}
```

**色彩方案必须包含**：
- Primary 色板（5 级：50/200/500/700/900）
- 语义色：Success/Warning/Danger/Info 各含背景和文字
- 中性灰：从 gray0 到 gray1000 的 11 级
- 语义背景色：页面背景、卡片背景、悬浮背景
- 语义文字色：主要文字、次要文字、辅助文字、禁用文字
- 在线状态色：在线（Success）、离开（Warning）、离线（gray400）、忙碌（Danger）

#### `typography.dart` — 字体层级

定义 10 级字体规格，每个含字号、字重、行高、字符间距：

| Token | 字号 | 字重 | 行高 | 用途 |
|-------|------|------|------|------|
| `headline1` | 28 | bold(700) | 1.3 | 页面大标题 |
| `headline2` | 24 | bold(700) | 1.3 | 弹窗标题 |
| `headline3` | 20 | semiBold(600) | 1.3 | 模块标题 |
| `title1` | 18 | semiBold(600) | 1.4 | 卡片标题 |
| `title2` | 16 | semiBold(600) | 1.4 | 列表项标题 |
| `body1` | 16 | regular(400) | 1.5 | 正文 |
| `body2` | 14 | regular(400) | 1.5 | 次要正文、消息正文 |
| `caption1` | 13 | regular(400) | 1.4 | 辅助说明 |
| `caption2` | 12 | regular(400) | 1.4 | 时间戳、角标 |
| `overline` | 11 | medium(500) | 1.3 | 标签、分类 |

跨平台字体族：中文使用系统默认（PingFang SC / Microsoft YaHei），英文和数字使用系统默认（SF Pro / Segoe UI）。等宽字体用于代码块：`SF Mono / Cascadia Code / monospace`。

#### `spacing.dart` — 间距规范

基于 8px 网格系统，提供 xs(4) / sm(8) / md(12) / base(16) / lg(20) / xl(24) / xxl(32) / xxxl(48) 共 8 级间距。

页面外边距（horizontal padding）：移动端 16px、平板 24px、桌面 32px。

#### `radius.dart` — 圆角体系

| Token | 值 | 用途 |
|-------|-----|------|
| `radiusNone` | 0 | 表格、分隔线 |
| `radiusSm` | 4 | 小标签、角标 |
| `radiusMd` | 8 | 输入框、按钮、卡片、消息气泡 |
| `radiusLg` | 12 | 大卡片、弹窗 |
| `radiusXl` | 16 | 底部弹出面板 |
| `radiusFull` | 999 | 头像、药丸形标签 |

#### `shadows.dart` — 阴影层级

3 级阴影，参考 Material Elevation：

| Token | 用途 |
|-------|------|
| `shadowSm` | 卡片（悬浮态）、下拉菜单 |
| `shadowMd` | 弹窗、底部面板、FAB |
| `shadowLg` | 模态框 |

每个阴影含 color、offset、blurRadius、spreadRadius 四个属性。

#### `icons.dart` — 图标映射

将项目中使用的图标统一映射到 Material Icons 或自定义 SVG。定义 `AppIcons` 类，包含至少 30 个图标常量：

- 导航：聊天、联系人、协作、日程、设置
- 操作：搜索、添加、更多、编辑、删除、复制、分享、下载
- 状态：在线、离线、通知、锁定、已读、未读
- 文件：文档、表格、看板、审批、图片、附件
- VE：机器人、AI、工具

### 2. 主题文件

#### `app_theme.dart` — Flutter ThemeData 封装

```dart
class AppTheme {
  static ThemeData light() {
    return ThemeData(
      useMaterial3: true,
      colorScheme: ColorScheme.light(
        primary: AppTokens.colorPrimary,
        // ... 完整映射
      ),
      scaffoldBackgroundColor: AppTokens.bgPrimary,
      appBarTheme: AppBarTheme(
        backgroundColor: AppTokens.bgPrimary,
        surfaceTintColor: Colors.transparent,
        elevation: 0,
        scrolledUnderElevation: 0.5,
        centerTitle: false,
        titleTextStyle: AppTypography.title2.style,
      ),
      bottomNavigationBarTheme: BottomNavigationBarThemeData(
        type: BottomNavigationBarType.fixed,
        backgroundColor: AppTokens.bgPrimary,
        selectedItemColor: AppTokens.colorPrimary,
        unselectedItemColor: AppTokens.textTertiary,
        elevation: 0,
      ),
      cardTheme: CardTheme(
        color: AppTokens.bgPrimary,
        elevation: 0,
        shape: RoundedRectangleBorder(
          borderRadius: BorderRadius.circular(AppTokens.radiusMd),
          side: BorderSide(color: AppTokens.borderLight),
        ),
      ),
      inputDecorationTheme: InputDecorationTheme(
        filled: true,
        fillColor: AppTokens.bgSecondary,
        contentPadding: EdgeInsets.symmetric(horizontal: 16, vertical: 12),
        border: OutlineInputBorder(
          borderRadius: BorderRadius.circular(AppTokens.radiusMd),
          borderSide: BorderSide.none,
        ),
        focusedBorder: OutlineInputBorder(
          borderRadius: BorderRadius.circular(AppTokens.radiusMd),
          borderSide: BorderSide(color: AppTokens.colorPrimary, width: 1.5),
        ),
      ),
      dividerTheme: DividerThemeData(
        color: AppTokens.borderLight,
        thickness: 0.5,
        space: 0,
      ),
      // ... 其余组件主题
    );
  }
}
```

每个 Flutter 组件类别都需要主题覆盖：AppBar、BottomNav、Card、Input、Button、Chip、Dialog、SnackBar、TabBar、ListTile、FloatingActionButton。

### 3. 全局组件库

创建 `apps/flutter/lib/design/widgets/` 目录，包含以下公共组件：

#### `app_button.dart`
三种层级按钮：
- `AppButton.primary(label, onPressed)` — 主色填充
- `AppButton.secondary(label, onPressed)` — 主色描边
- `AppButton.text(label, onPressed)` — 纯文字

支持 `size: small/medium/large`、`loading: bool`、`disabled: bool`、`icon: IconData?`。

#### `app_avatar.dart`
头像组件，支持：
- 用户头像（显示首字母 + 背景色）
- VE 头像（机器人图标 + 主色渐变背景）
- 在线状态指示器（右下角小圆点）
- 尺寸：xs(24)/sm(32)/md(40)/lg(56)/xl(80)

```dart
AppAvatar.user(name: "张三", size: AvatarSize.md, status: OnlineStatus.online)
AppAvatar.ve(name: "虚拟员工", size: AvatarSize.md)
```

#### `app_empty_state.dart`
空状态占位组件：
```dart
AppEmptyState(
  icon: AppIcons.chat,
  title: "暂无消息",
  description: "开始与团队成员对话吧",
  action: AppButton.primary("发起聊天", onPressed: () {}),
)
```

#### `app_loading.dart`
加载指示器组件：全屏加载、局部加载、下拉刷新、上拉加载更多。

#### `app_error_widget.dart`
错误状态组件：错误图标 + 错误信息 + 重试按钮。

#### `app_badge.dart`
角标组件：未读消息数（支持 99+）、在线状态点、审批状态标签。

## 约束 (Constraints)

- 所有新 UI 代码必须使用本单元定义的设计 Token，禁止硬编码颜色值、字号、间距
- 设计 Token 文件独立，不依赖任何业务逻辑
- 每个公共组件含完整的 `///` 文档注释
- 现有业务逻辑不修改，只替换视觉效果
