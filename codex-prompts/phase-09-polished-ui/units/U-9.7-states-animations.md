# U-9.7 空状态、加载、错误处理 + 页面动效与微交互

## 目标 (Goal)

为全应用统一三类反馈状态（空状态/加载态/错误态）的视觉表现，并添加页面过渡动效和微交互，让应用在"没有数据"和"等待中"时同样专业。

## 上下文 (Context)

- 前置：U-9.6（协作工具 UI 完成）
- 当前应用缺少统一的空状态/加载/错误组件，页面切换无过渡动画

## 工作范围

### 1. 空状态体系

全局搜索所有使用"空列表"或"无数据显示"的场景，统一替换为 `AppEmptyState` 组件（已在 U-9.1 定义）。每个场景提供独特的图标和文案：

| 场景 | 图标 | 标题 | 描述 | 操作按钮 |
|------|------|------|------|---------|
| 无消息 | `chat_bubble_outline` | 暂无消息 | 开始与团队成员对话吧 | 发起聊天 |
| 无联系人 | `people_outline` | 暂无联系人 | 邀请成员加入组织 | 邀请成员 |
| 无频道 | `tag` | 暂无频道 | 创建一个频道开始协作 | 创建频道 |
| 无文档 | `article_outline` | 暂无文档 | 创建协作文档 | 新建文档 |
| 无表格 | `table_chart_outline` | 暂无表格 | 创建多维表格 | 新建表格 |
| 无看板 | `view_kanban_outline` | 暂无看板 | 创建任务看板 | 新建看板 |
| 无审批 | `approval_outline` | 暂无审批 | 发起审批流程 | 发起审批（或隐藏按钮） |
| 无日程 | `calendar_today_outline` | 暂无日程 | 创建日程事件 | 新建日程 |
| 无搜索结果 | `search_off_outline` | 未找到相关结果 | 尝试其他关键词 | — |
| 无 VE | `smart_toy_outline` | 暂无虚拟员工 | 创建你的第一个 AI 助手 | 创建 VE |
| 无组织 | `corporate_fare_outline` | 暂无组织 | 创建或加入一个组织 | 创建组织 |
| 网络错误 | `cloud_off_outline` | 连接失败 | 请检查网络后重试 | 重试 |
| 服务器错误 | `error_outline_outline` | 服务异常 | 服务器暂时不可用 | 重试 |

### 2. 加载状态体系

| 场景 | 组件 | 说明 |
|------|------|------|
| 页面首次加载 | `AppLoading.fullPage(message: "加载中...")` | 居中 `CircularProgressIndicator` + 可选文字 |
| 列表加载更多 | 列表底部 `Center(child: CircularProgressIndicator(strokeWidth: 2))` | 高度 48px，居中小圈 |
| 按钮提交中 | `AppButton.primary("提交中...", loading: true, onPressed: null)` | 按钮内部小圈 + 禁用 |
| 下拉刷新 | `RefreshIndicator`（Material 标准） | 主色刷新指示器 |
| 图片加载中 | `CachedNetworkImage` 的 `placeholder`：灰色骨架屏 | 含淡入动画 |
| 骨架屏 | `Shimmer` 效果（可选，使用 `shimmer` package） | 频道列表、消息列表首次加载 |

**骨架屏规范**：
- 频道列表骨架：圆形头像占位(40px) + 两行文字占位 + 时间占位
- 消息列表骨架：左侧小圆 + 两行文字（不同宽度，模拟真实消息长度）
- 骨架动画：从左到右的高光扫过（`shimmer` package），持续时间 1.5s

### 3. 错误状态体系

| 场景 | 组件 | 行为 |
|------|------|------|
| 网络错误 | `AppErrorWidget.network(onRetry:)` | "网络连接失败" + 重试按钮 |
| 服务器错误(5xx) | `AppErrorWidget.server(onRetry:)` | "服务器暂时不可用" + 重试按钮 |
| 权限错误(403) | `AppErrorWidget.permission()` | "你没有权限访问" |
| 页面不存在(404) | `AppErrorWidget.notFound()` | "页面不存在" + 返回首页按钮 |
| 一般错误 | `AppErrorWidget.general(message, onRetry:)` | 自定义错误消息 + 重试 |

所有错误组件包含：错误图标（顶部，48px）+ 错误标题（`title2`）+ 错误描述（`body2`，`textSecondary`）+ 操作按钮。

### 4. 页面过渡动效

修改 GoRouter 配置，为所有页面路由添加过渡动画：

| 动画 | 场景 | 效果 |
|------|------|------|
| 水平滑入 | 列表→详情页（频道→聊天、通讯录→个人、文档列表→编辑器） | 从右滑入（300ms，`Curves.easeOutCubic`） |
| 垂直弹出 | 模态操作（创建频道、创建文档、审批决策） | 从底部弹出（250ms，`Curves.easeOutCubic`） |
| 淡入 | 设置页、关于页 | 淡入（200ms） |

返回时反向动画（`ReverseAnimation`）。

### 5. 微交互

| 交互 | 触发 | 效果 |
|------|------|------|
| 按钮按下 | `onTapDown` | 轻微缩放（`scale: 0.97`，100ms） |
| 列表项点击 | `onTap` | `InkWell` 涟漪效果（Material 默认） |
| 发送消息 | 点击发送 | 消息气泡从底部弹入（`SlideTransition` + `FadeTransition`，200ms） |
| 未读→已读 | 查看消息 | 未读角标数字平滑消失（`SizeTransition`） |
| 反应添加 | 长按→选 emoji | emoji 弹跳动画（`ScaleTransition`，从 0→1.3→1.0，300ms，`Curves.elasticOut`） |
| 删除操作 | 滑动删除 | 被删除项向右滑出 + 淡出（200ms） |
| 标签切换 | 点击 BottomNav/BottomTab | 选中图标微弹跳（`ScaleTransition`，0.8→1.0，200ms） |
| 开关切换 | `Switch`/`Checkbox` | Material 默认动画 |

使用 `AnimationController` 或 `AnimatedContainer`/`AnimatedOpacity` 等隐式动画组件实现。

### 6. 列表动画

- 新消息到达：列表底部新行淡入（`FadeTransition`，100ms）
- 列表刷新：现有内容淡出→新内容淡入（交叉淡入淡出，`AnimatedSwitcher`）
- 列表项删除：`Dismissible` widget 的滑动删除动画

## 平台动效差异（遵循 U-9.2 平台规范）

| 动效 | 移动端 | 桌面端 |
|------|--------|--------|
| 页面切换 | 水平滑入（300ms，`Curves.easeOutCubic`），iOS 用 `CupertinoPageRoute` | 淡入淡出（200ms，`Curves.easeInOut`），更克制 |
| 弹窗出现 | 底部滑入（250ms） | 居中缩放弹出（`ScaleTransition` 0.9→1.0，200ms） |
| 发送消息 | 气泡弹入（`ScaleTransition` + `SlideTransition`，200ms） | 淡入（150ms），不弹跳 |
| 列表项删除 | 侧滑动画（`Dismissible`） | 淡出收缩（`SizeTransition` + `FadeTransition`） |
| hover 效果 | 无 | 背景色平滑过渡（`AnimatedContainer`，100ms） |
| 滚动 | 惯性滚动（物理模拟） | 精确滚动（无惯性） |
| 回弹效果 | iOS 底部列表过度滚动回弹（`BouncingScrollPhysics`） | 无（`ClampingScrollPhysics`） |

**实现要求**：
- 使用 `Theme.of(context).platform` 判断当前平台选择对应的动效风格
- iOS 平台全局使用 `CupertinoPageTransitionsBuilder`
- 桌面端动画时长不超过 200ms（保持效率感），移动端不超过 300ms

## 约束 (Constraints)

- 所有动画时长在 100-300ms 之间，桌面端不超过 200ms
- 使用 Material 内置动画 Curve，不自定义复杂曲线
- 骨架屏使用 `shimmer` package（`flutter pub add shimmer`）
- 错误重试按钮调用原页面的数据加载方法
- 严格遵守 U-9.2 定义的平台惯例规则
