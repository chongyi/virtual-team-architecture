# U-9.2 应用壳与导航体系

## 目标 (Goal)

建立响应式应用壳（App Shell），包含：移动端底部标签导航、平板/桌面端侧边栏导航、全局顶栏（含搜索和用户菜单），实现导航框架的跨端统一体验。

## 上下文 (Context)

- 前置：U-9.1（设计系统完成）
- 当前 Flutter 路由使用 go_router，导航状态散落在各页面
- 需要统一的应用壳来承载所有页面内容

## 工作范围

### 1. 响应式布局断点

在 `apps/flutter/lib/design/layout.dart` 中定义断点：

| 断点 | 宽度 | 导航模式 | 内容区 |
|------|------|---------|--------|
| Mobile | < 600 | 底部标签导航（3-4 项） | 全宽，无侧边栏 |
| Tablet | 600-1023 | 底部标签 + 可选侧边栏 | 侧边栏可收折 |
| Desktop | >= 1024 | 左侧固定导航栏 | 侧边栏常驻 + 内容区 |

### 2. 移动端底部导航

`apps/flutter/lib/app/shell/mobile_shell.dart`

底部导航栏含 4 个标签（Tab），图标+文字，选中态高亮：

| 位置 | 标签 | 图标 | 路由 |
|------|------|------|------|
| 1 | 消息 | `chat_bubble_outline` / `chat_bubble` | `/im` |
| 2 | 通讯录 | `contacts_outline` / `contacts` | `/contacts` |
| 3 | 协作 | `grid_view_outline` / `grid_view` | `/tools` |
| 4 | 我 | `person_outline` / `person` | `/profile` |

特性：
- 未读消息角标在"消息"标签右上角（数字或红点）
- 切换标签时保持页面状态（IndexedStack 或 AutomaticKeepAliveClientMixin）
- 导航栏背景使用 `AppTokens.bgPrimary`，顶部细分割线

### 3. 桌面端侧边栏

`apps/flutter/lib/app/shell/desktop_shell.dart`

左侧固定宽度 260px 的侧边栏，垂直排列：

**顶部区域**：
- 应用 Logo/名称（"Virtual Team" 或自定义 Logo）
- 当前租户/组织名称（可点击切换）

**导航区域**（垂直列表）：
- 首页/消息 — icon + "消息"
- 通讯录 — icon + "通讯录"
- 协作工具 — icon + "协作"（可展开子项：文档/表格/看板/审批/日程）
- 设置 — icon + "设置"

**底部区域**：
- 用户头像（小）+ 名称 + 在线状态
- 点击展开用户菜单（个人设置 / 切换租户 / 退出登录）

每个导航项 hover 时背景变 `AppTokens.bgHover`，选中项左侧高亮条（3px Primary 色）。

侧边栏与内容区之间 1px 分割线（`AppTokens.borderLight`）。

### 4. 全局顶栏

`apps/flutter/lib/app/shell/app_bar.dart`

移动端顶栏（在消息/通讯录等列表页展示）：
- 左侧：页面标题（如"消息"、"通讯录"）
- 右侧：搜索图标（点击展开搜索栏）+ 更多操作图标

桌面端顶栏：
- 左侧：面包屑或当前页面标题
- 中间：全局搜索输入框（可选，仅在需要时显示）
- 右侧：通知图标（铃铛）+ 用户头像

顶栏高度：移动端 56px，桌面端 64px。背景白色，底部可选分割线。

### 5. AppShell 入口

`apps/flutter/lib/app/shell/app_shell.dart`

使用 `LayoutBuilder` 判断当前断点，对移动端渲染 `MobileShell`，对平板/桌面渲染 `DesktopShell`。

```dart
class AppShell extends StatelessWidget {
  final Widget child; // 当前路由页面内容
  // ...
  Widget build(BuildContext context) {
    return LayoutBuilder(builder: (context, constraints) {
      if (constraints.maxWidth < 600) {
        return MobileShell(child: child);
      } else {
        return DesktopShell(child: child);
      }
    });
  }
}
```

### 6. 路由重构

修改 `routes.dart`，将顶层路由包裹在 `AppShell` 中：
- `ShellRoute` 模式：`/im`、`/contacts`、`/tools`、`/profile` 包在 AppShell 内
- 非壳路由（登录、设置详情、频道详情）不加壳

## 约束 (Constraints)

- 使用 U-9.1 定义的全部设计 Token，禁止硬编码颜色/字号/间距
- 所有导航项图标使用 `AppIcons` 中定义的常量
- 页面状态保持：标签切换不丢失滚动位置和输入内容
- 支持键盘快捷键（桌面端）：`Cmd+K` 打开搜索、`Cmd+1/2/3/4` 切换标签

## 平台差异规范（关键：本单元定义全应用遵守的平台规则）

以下规则适用于 Phase 9 的所有后续单元，每个单元必须遵守，不得单独处理平台差异。

### 一、输入方式差异

| 场景 | 移动端（触控） | 桌面端（鼠标+键盘） |
|------|--------------|-------------------|
| 主操作 | 点击（Tap） | 点击（Click） |
| 次要操作 | 长按（Long Press）+ 触觉反馈 | 右键菜单（ContextMenu）或 hover 显示操作按钮 |
| 选择文本 | 长按选中 + 系统选择手柄 | 双击选词 / 三击选段 / 拖拽选择 |
| 滚动 | 手指滑动（惯性滚动） | 滚轮 + 滚动条（`Scrollbar` 常驻） |
| 列表项操作 | 左右滑动（Dismissible） | hover 显示操作图标按钮，或右键菜单 |
| 返回上级 | 左边缘右滑手势 / Android 返回键 | 顶栏返回按钮 / `Escape` 键 |
| 拖拽排序 | 长按触发拖拽（`LongPressDraggable`） | 鼠标拖拽（`Draggable` 直接响应） |
| 多选 | 长按进入多选模式 | `Cmd/Ctrl + Click` 或 `Shift + Click` 范围选择 |

**实现要求**：
- 所有可交互元素在桌面端必须有 `MouseCursor` 样式（`SystemMouseCursors.click` 用于可点击、`SystemMouseCursors.text` 用于可选中）
- 所有列表项在桌面端 hover 时显示背景变化（`bgHover`）+ 可选操作图标（如频道列表的置顶/退出按钮平时隐藏、hover 显示）
- 所有 `IconButton` 和可点击区域在桌面端有 `tooltip`
- 移动端的长按操作统一附带 `HapticFeedback.lightImpact()` 触觉反馈
- 桌面端滚动区域统一包裹 `Scrollbar`（`thumbVisibility: true`），移动端依靠系统滚动指示器

### 二、布局密度差异

同一页面在移动端和桌面端的信息密度不同：

| 属性 | 移动端 | 平板 | 桌面端 |
|------|--------|------|--------|
| 页面水平内边距 | 16px | 24px | 32px |
| 列表项高度 | 56px（标准）/ 72px（含描述） | 52px / 64px | 48px / 60px |
| 卡片圆角 | 8px（`radiusMd`） | 10px | 12px（`radiusLg`） |
| 消息气泡最大宽度 | 80% | 70% | 60% |
| 网格列数（文档/表格列表） | 2 列 | 3 列 | 4 列 |
| 弹窗宽度 | 全屏或 90% 底部面板 | 480px 居中 | 560px 居中 |
| 字体缩放 | 跟随系统 | 跟随系统 | 跟随系统 |

**实现要求**：
- 使用 `LayoutBuilder` 或 `MediaQuery.of(context).size.width` 判断当前断点
- 页面内边距通过 `AppTokens.pageHorizontalPadding(context)` 方法获取（内部判断断点返回对应值）
- 列表项高度通过 `AppTokens.listTileHeight(context, hasDescription: true/false)` 获取
- 弹窗使用 `showModalBottomSheet`（移动端）+ `showDialog`（桌面端），统一封装为 `AppDialogs.show(...)` 方法

### 三、导航与页面切换差异

| 场景 | 移动端 | 桌面端 |
|------|--------|--------|
| 主导航 | 底部标签导航栏（固定 56px 高度） | 左侧导航栏（固定 260px 宽度） |
| 返回按钮 | AppBar 左侧（自动 `BackButton`） | 内容区顶栏左侧 + `Escape` 快捷键 |
| 深层导航 | 逐页 push，全屏过渡 | 右侧内容区切换（不一定要 push），或使用面包屑 |
| 搜索入口 | 点击顶栏搜索图标 → push 全屏搜索页 | `Cmd+K` 快捷键 → 弹出搜索浮层（类似 Spotlight） |
| 快捷操作 | FAB 或底部浮动按钮 | 顶栏按钮或右键菜单 |
| 页面切换动画 | 水平滑入（SlideTransition，300ms） | 淡入或微缩放（200ms，更克制） |

**实现要求**：
- 桌面端搜索浮层：覆盖式半透明蒙层 + 居中搜索框 + 即时结果，无页面跳转
- 桌面端右键菜单使用 `showMenu` 或 `ContextMenu` widget
- 移动端不使用面包屑，桌面端在内容区顶栏显示面包屑（全局面包屑状态由 `Riverpod` 管理）

### 四、弹窗与对话框差异

| 场景 | 移动端 | 桌面端 |
|------|--------|--------|
| 简单确认 | `showDialog`（AlertDialog） | 同 |
| 表单/复杂操作 | 底部面板（`showModalBottomSheet`，90% 高度） | 居中弹窗（`showDialog`，560px 宽） |
| 选择器（日期/时间/下拉） | 系统原生弹出（`showDatePicker` 等） | 内联或下拉浮层（DropdownMenu） |
| SnackBar/Toast | 屏幕底部 | 右下角 |
| 工具提示（Tooltip） | 无需（无 hover） | 所有图标按钮必须有 |

**实现要求**：
- 封装 `AppDialogs` 工具类，提供统一的弹窗入口：
  - `AppDialogs.confirm(context, title, message, onConfirm)` — 基础确认
  - `AppDialogs.form(context, title, child, onSubmit)` — 表单弹窗（自动判断平台选择底部面板或居中弹窗）
  - `AppDialogs.actionSheet(context, title, actions)` — 操作面板
- 移动端的底部面板保留 20px 顶部拖拽手柄（DragHandle），桌面端的居中弹窗无手柄

### 五、平台惯例差异

| 平台 | 设计语言 | 特殊要求 |
|------|---------|---------|
| iOS | Cupertino | 左边缘右滑返回全应用启用、日期选择器默认 iOS 风格、开关用 CupertinoSwitch |
| Android | Material 3 | 系统返回键处理（`PopScope`）、Material You 动态色彩（已覆盖） |
| macOS | 桌面 | 菜单栏集成（可选）、`Cmd` 快捷键体系、窗口关闭不退出应用 |
| Windows | 桌面 | 窗口最小尺寸 400x300、标题栏图标 |
| Web | 浏览器 | 响应式布局（已覆盖）、PWA 离线支持（可选）、浏览器通知替代推送 |

**实现要求**：
- iOS 平台：`ThemeData` 的 `pageTransitionsTheme` 使用 `CupertinoPageTransitionsBuilder`（水平滑入返回手势）
- macOS 平台：快捷键使用 `Cmd`（`Meta`）修饰键，Windows/Linux 使用 `Ctrl`
- 使用 `defaultTargetPlatform` 或 `Theme.of(context).platform` 判断当前平台
- 平台判断逻辑集中在 `AppPlatform` 工具类中（`lib/core/platform/platform_info.dart`），不散落在业务代码中

### 六、特殊设备适配

| 设备类型 | 特殊处理 |
|---------|---------|
| 平板横屏 | 强制使用侧边栏 + 内容区双栏布局（即使宽度 < 1024） |
| 折叠屏 | 折叠态用移动端、展开态用桌面端（监听 `MediaQuery` 变化实时切换） |
| 小屏手机（< 360px） | 进一步压缩内边距至 12px、列表项高度 48px、隐藏次要信息 |
| 大屏桌面（> 1600px） | 内容区最大宽度 960px 居中（防止行过长），两侧留白 |

**实现要求**：
- 平板横屏判断：`width >= 768 && MediaQuery.of(context).orientation == Orientation.landscape` — 此时强制使用桌面端布局
- 折叠屏/大屏使用 `ConstrainedBox` 限制内容最大宽度
- 小屏使用 `AppTokens.compact()` 返回更紧凑的间距 Token 集
