# U-8.1 SeaORM 迁移全面完成

## 目标 (Goal)

## 工作范围

### 第一阶段：补全 Entity 定义（migration crate）

在 `migration/src/` 下新建 7 个迁移文件，每个含 `#[derive(DeriveEntityModel)]` 的 Entity + 对应的 Migration：

| 文件 | Entity | 对应 raw SQL |
|------|--------|-------------|
| `m20260519_100000_documents.rs` | DocumentEntity、DocumentRevisionEntity | 0010 + 0016 |
| `m20260519_100001_bitables.rs` | BitableTableEntity、BitableColumnEntity、BitableRowEntity | 0011 |
| `m20260519_100002_boards.rs` | BoardEntity、BoardColumnEntity、BoardCardEntity | 0012 |
| `m20260519_100003_approvals.rs` | ApprovalTemplateEntity、ApprovalInstanceEntity、ApprovalHistoryEntity | 0013 |
| `m20260519_100004_schedule.rs` | CalendarEventEntity、ReminderEntity | 0014 |
| `m20260519_100005_audit.rs` | AuditEventEntity | 0015 |
| `m20260519_100006_wen_nodes.rs` | WenNodeEntity | agent-server 内联 DDL |

每个 Entity 要求：
- `table_name` 与现有 PG 表名一致
- 列类型、约束、默认值与 raw SQL 完全一致
- Relation 定义（`belongs_to`、`has_many`）按外键关系链接
- CHECK 约束通过 `#[sea_orm(indexed)]`、`#[sea_orm(unique)]` 等属性表达
- 导出到 `migration/src/lib.rs` 的 `pub mod entities` 块

### 第二阶段：重写 collab-server store（11 个文件）

将 `crates/collab-server/src/store/` 下所有 store 从 `sqlx::query()` 迁移到 SeaORM 查询模式：

| 文件 | 需要迁移的查询 |
|------|--------------|
| `message.rs` | insert、list（分页）、get_by_id、update、soft_delete、search（FTS 保留 tsvector） |
| `channel.rs` | create、list、get_by_id、add_member、remove_member |
| `document.rs` | create、list、get_by_id、update（乐观锁）、archive、list_revisions |
| `bitable.rs` | create_table、list_tables、add_column、create_row、update_row、query_rows（视图过滤） |
| `board.rs` | create_board、list_boards、add_column、create_card、move_card、list_swimlanes |
| `approval.rs` | create_template、create_instance、update_instance、list_instances、record_history |
| `schedule.rs` | create_event、list_events、update_event、cancel_event、create_reminder、trigger_reminder |
| `org.rs` | create_org、list_orgs、get_org、update_org、archive_org、add_member、remove_member |
| `user.rs` | create_user、find_by_display_name、list_users |
| `reaction.rs` | toggle_reaction、delete_reaction |
| `attachment.rs` | insert_attachment、get_attachment、list_attachments |

**查询模式转换规则**：
- `sqlx::query("INSERT INTO ...")` → `Entity::insert(active_model).exec(db).await`
- `sqlx::query("SELECT ... WHERE ...")` → `Entity::find().filter(...).all(db).await`
- `sqlx::query("UPDATE ... SET ...")` → `ActiveModel::set(...).update(db).await`
- `sqlx::query("DELETE ...")` → `Entity::delete_by_id(id).exec(db).await`
- 全文搜索（messages.search_vector）保留 `sqlx::query()` — SeaORM 不提供 FTS 支持
- 批量操作使用 `Entity::insert_many()`
- 乐观锁（document version）通过 `ActiveModel::update_with_condition()` 实现

**Store trait 不变**：`DocumentStore`、`MessageStore` 等 trait 接口保持不变。只替换 trait 实现内部的 SQL 执行方式。

### 第三阶段：重写 agent-server store（3 个文件）

| 文件 | 需要迁移的查询 |
|------|--------------|
| `management/store.rs` | VE instance CRUD（`InstanceRow` → `VeInstanceEntity`） |
| `management/node_registry.rs` | WEN node 持久化（`PgNodeRegistryStore` → `WenNodeEntity`） |
| `ve/work_context_store.rs` | WorkContext CRUD（`WorkContextRow` → `WorkContextEntity`） |

同时删除 `migrate()` 函数中的内联 `CREATE TABLE IF NOT EXISTS`，改用 `sea_orm_migration::MigratorTrait` 统一迁移。

### 第四阶段：清理与统一

| 操作 | 说明 |
|------|------|
| 删除 `/migrations/` 目录（16 个 raw SQL） | 已全部由 SeaORM Rust 迁移替代 |
| 删除 `collab-server/src/db.rs` 中的死函数 | `sea_orm_connection_from_pool()` |
| collab-server `AppState` 增加 `sea_orm::DatabaseConnection` | 复用同一 `sqlx::PgPool` 构建，注入各 store |
| collab-server store 模块增加 `pub fn new(db: DatabaseConnection) -> Self` | 统一使用 SeaORM 连接 |
| collab-server 移除 `sqlx` 直接依赖 | 如全部迁移完成则无需 sqlx（保留 FTS 那一路除外） |
| agent-server 移除内联 DDL 的 `migrate()` | 统一由 migration crate 管理 |
| vta/store-sqlite 保持 `sqlx::sqlite` | SQLite 侧不迁移（SeaORM 支持 sqlite 但 VTA session 存储用 sqlx 足够轻量） |

## 约束 (Constraints)

- **所有现有测试必须继续通过**（约 140+ 测试），不得改变任何 API 行为
- SeaORM `DatabaseConnection` 复用现有 `sqlx::PgPool`，不创建新的连接池
- 全文搜索（messages FTS）保留 sqlx — SeaORM 不支持 PostgreSQL `tsvector`/`ts_query`
- Store trait 接口不变，只替换实现
- 迁移文件命名遵循 SeaORM CLI 格式：`mYYYYMMDD_HHMMSS_description.rs`
- 新 Entity 的 `table_name` 与当前 PG 表名完全一致（蛇形命名）
