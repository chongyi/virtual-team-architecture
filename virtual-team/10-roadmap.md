# 10. 实施路线与 VTA 迭代需求

## 1. 路线图原则

本文档描述完整目标架构。实施路线只是工程落地顺序，不代表目标模型可以被简化成单会话 Agent 或单纯聊天机器人。

每个阶段都应服务于最终形态：

```text
VT 项目
  → 虚拟团队应用
  → 服务端架构
  → 消息网关
  → AgentServer
  → WorkEnvironmentNode
  → VTE
  → VTA
```

## 2. Phase 0：文档与概念冻结

目标：

- 冻结 VT 顶层概念。
- 明确 VT 与 VTA 边界。
- 明确虚拟团队应用不是 Agent UI 外壳。
- 明确 VTE、AgentServer、WorkEnvironmentNode、工作上下文的职责。
- 形成可供后续设计和开发引用的文档体系。

关键产物：

- 本目录文档。
- 术语表。
- 顶层架构图。
- 服务边界说明。
- VTA 迭代需求列表。

## 3. Phase 1：虚拟团队应用基础

目标是让协作应用本身成立。

能力范围：

- 租户、用户、组织。
- 成员和角色。
- 私聊、群聊、频道。
- Append-only 消息。
- 文件和附件。
- 基础任务或事项视图。
- 审批入口的产品形态。
- 虚拟员工成员类型预留。

这一阶段即使没有 AgentServer，也应该是一个可理解的通讯协作工具。

## 4. Phase 2：AgentServer 与基础 VTE

目标是让虚拟员工以成员身份接入应用。

能力范围：

- AgentServer 基础服务。
- 虚拟员工注册、绑定和状态。
- 默认私人小助理。
- 应用消息路由到 VTE。
- VTE 消息前处理和后处理。
- 工作上下文创建、关联和状态。
- 服务端轻量工具。
- VTA Session 创建和恢复。
- 基础权限与审计。

关键验证：

- 用户能和私人小助理对话。
- 助理能引导创建组织和虚拟员工。
- 明确任务能形成工作上下文。
- 简单任务能完成并在聊天中汇报。

## 5. Phase 3：WorkEnvironmentNode

目标是让虚拟员工使用真实工具环境工作。

能力范围：

- 本地工作环境客户端。
- 节点注册、连接和心跳。
- 工具清单上报。
- 虚拟员工与节点绑定。
- 隔离空间。
- 工具调用转发。
- 审批策略。
- 执行结果和文件变更回传。
- 节点离线和失败处理。

关键验证：

- 一个虚拟员工可以使用本地节点工具。
- 一个设备可以提供多个隔离空间。
- 工具风险操作需要审批。
- 没有节点时，VTE 能提出配置需求。

## 6. Phase 4：完整工作上下文与多员工协作

目标是支撑复杂任务和多虚拟员工协作。

能力范围：

- 工作上下文 fork。
- 子 Agent。
- 多 VTE 协作。
- 管理型虚拟员工。
- 任务拆分、分派、汇总和审查。
- 交付物管理。
- 跨频道、跨组织任务引用。
- 更强上下文压缩和长期记忆。

关键验证：

- 一个复杂任务可由助理拆分给多个虚拟员工。
- 不同虚拟员工在同一工作上下文下协作。
- 用户可以在聊天中随时插入补充信息。
- 系统能正确关联或 fork 工作上下文。

## 7. Phase 5：云工作环境与商业化

目标是形成商业闭环。

能力范围：

- 云 WorkEnvironmentNode。
- 环境规格和套餐。
- 高级助理。
- 高级虚拟员工。
- 商业化服务端工具。
- 用量和计费。
- 企业权限、审计和 SLA。
- 工具和虚拟员工市场雏形。

## 8. VTA 迭代需求

从 VT 目标反推，VTA 需要关注以下能力：

### 8.1 租户隔离

VTA Store、Session、Message、Event、Tool 调用都需要能够接收并维持租户上下文。

### 8.2 RemoteToolExecutor

VTA 需要支持远程工具执行抽象，让工具可以由 WorkEnvironmentNode 执行并异步回传结果。

### 8.3 Tool Approval Continuation

工具调用被审批中断后，需要能恢复原 Agent loop，而不是让上层手动拼接状态。

### 8.4 Work Context 关联

VTA 不需要实现 VT 的工作上下文产品模型，但需要允许 Session、Turn、Message 携带外部 work_context_id 或 metadata，以便 AgentServer 建立关联。

### 8.5 子 Agent 可观测性

VTA 已采用独立 Session + parent 链接模型。VT 需要在此基础上把子 Agent 状态映射到工作上下文和产品视图。

### 8.6 Prompt 动态注入

VTE 需要把组织、频道、成员、权限、工作上下文和工具环境注入 prompt。VTA PromptManager 需要支持稳定的模板和动态上下文组合。

### 8.7 配置包版本治理

虚拟员工的岗位定义是长期资产，需要配置包版本、审计和回滚。

### 8.8 工具能力投影

VTA 需要能把 AgentServer 和 WorkEnvironmentNode 提供的工具能力投影给模型，同时避免工具列表过载。
