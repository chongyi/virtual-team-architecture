## 可观测性标准

- [ ] 操作入口有 INFO 级别日志（含 tenant_id、actor_id、request_id）
- [ ] 异常路径有 ERROR 级别日志（含错误码和上下文）
- [ ] 关键操作有 metrics counter
- [ ] 外部调用有 latency histogram
- [ ] 跨服务调用有 tracing span（含 correlation_id）

## 权限与安全标准

- [ ] tenant_id 隔离已验证（跨租户访问被拒绝）
- [ ] 越权操作被正确拒绝并记录审计日志
- [ ] 用户可见错误不泄露内部信息（堆栈、路径、数据库错误）
- [ ] 输入校验边界明确（API 层、服务层）

## 就绪定义 (Definition of Ready)

功能验收标准可进入开发阶段的前提：

- [ ] 所有核心验收场景已定义
- [ ] 重要的边界情况已覆盖
- [ ] 性能标准可测量
- [ ] 降级标准明确了依赖不可用时的行为
- [ ] 不与其他功能的验收标准冲突
