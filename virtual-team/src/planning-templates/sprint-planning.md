# Sprint 规划模板

> 此模板用于基于本设计指南构建具体开发 Sprint 的计划。具体的 Sprint 计划、任务状态和进度跟踪应独立维护在项目管理工具中（如 Linear、Jira、GitHub Projects）。

## Sprint 信息

| 字段 | 内容 |
|------|------|
| **Sprint 编号** | Sprint-XX |
| **起止日期** | YYYY-MM-DD ~ YYYY-MM-DD |
| **持续时间** | 1-2 周 |
| **所属轨道** | A（Agent 核心链）/ B（协作应用）/ C（工作环境客户端） |
| **Sprint 目标** | 一句话描述本 Sprint 要达成的核心目标 |

## 协议边界

本 Sprint 涉及的协议边界（勾选）：

- [ ] 协作应用层协议
- [ ] 对接协议（协作应用 ↔ Agent 服务器）
- [ ] 内部协议（Agent 服务器 ↔ VE Runner ↔ WEN）
- [ ] VTA 核心 trait 接口
- [ ] 其他：______

若是此 Sprint 首次触及某协议边界，需在 Sprint 开始前确认相关协议文档已冻结。

## 验收标准

- [ ] 以下场景可端到端验证：（列场景）
- [ ] 降级行为与降级设计文档一致
- [ ] 幂等性验证通过（重复请求不产生副作用）
- [ ] 权限边界验证通过（越权请求被正确拒绝）
- [ ] 可观测性指标已接入（日志/指标/追踪）
