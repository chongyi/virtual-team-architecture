# 核心概念模型

本章定义 Virtual Team 的核心概念及其相互关系，作为后续所有架构设计的基础。

## 概念全景

```
用户 (User)
  │
  ├─ 创建/管理 ─→ 组织 (Organization)  ─  树状嵌套
  │                  │
  │                  └─ 包含 ─→ 虚拟员工 (Virtual Employee)
  │
  ├─ 拥有 ─→ 工作环境节点 (Work Environment Node)
  │              │
  │              └─ 分配给 ─→ 虚拟员工
  │
  └─ 通过 ─→ 协作应用 (Collaboration App)
                │
                ├─ 发送消息 ─→ 虚拟员工
                └─ 使用工具 ─→ 协作工具集
```

## 概念定义

### 用户 (User)

Virtual Team 的最终使用者。一个用户对应一个**租户（Tenant）**，拥有独立的数据空间和配置空间。

- 通过协作应用与虚拟员工交互
- 创建和管理组织
- 拥有并分配工作环境节点

### 组织 (Organization)

用户创建的逻辑管理单元，呈**树状结构**（组织下可嵌套子组织）。

- 用于数据和资源的逻辑划分
- 便于不同业务方向、客户或场景的分类管理
- 影响虚拟员工的调度——助理分析任务后，可按组织匹配合适的虚拟员工
- 每个组织可以指定 Leader 型虚拟员工作为管理者

大多数简单场景下，用户感知不到组织的存在。只有业务复杂度达到一定程度时，才需要显式创建多层组织。

### 虚拟员工 (Virtual Employee)

面向用户的一等 Agent 实体。在协作应用中表现为一个**联系人**，像真人同事一样接收消息、完成任务。

一个虚拟员工内部包含多个由 VTA 驱动的 Agent：

| 内部 Agent | 职责 |
|------------|------|
| **意图识别 Agent** | 分析每条消息的意图，判断是否需要新工作、关联现有工作上下文 |
| **主 Agent** | 实际执行工作，管理工作上下文，创建子 Agent |
| **子 Agent** | 主 Agent 动态创建，处理特定子任务 |

虚拟员工的能力由配置包定义：角色身份（prompt）、可用场景、模型选择、工具集、技能。

### 助理 (Assistant)

一种**特殊的虚拟员工**，没有底层架构特权。其不同之处仅在于配置包：

- 不同的 system prompt（管理型角色定位）
- 可能使用更强的模型（更好的分析协调能力）
- 可能拥有更多的组织管理类工具

用户首次进入应用时默认获得一个私人小助理，负责引导用户建立组织、招募虚拟员工、协助管理。

### 工作上下文 (Work Context)

虚拟员工处理一项具体工作任务时的**独立工作空间**。它是一个逻辑概念，内部由 VTA Session 支撑：

- **新工作 (New)**：创建全新的工作上下文，类似在 Claude Code 中打开新对话
- **Fork**：从已有工作的某个检查点分叉出新的工作上下文
- **Resume**：恢复已有工作上下文继续执行

工作上下文的意义在于：

1. 隔离不同任务的消息和上下文，避免污染
2. 支持独立的状态管理（暂停、恢复、归档）
3. 允许从工作中途分叉探索不同方案

### 工作环境节点 (Work Environment Node)

虚拟员工执行操作的实际物理/虚拟环境。用户安装运行工作环境客户端，该客户端连接服务端后，可分配给虚拟员工使用。

工作环境节点承载：

- MCP Server 接入
- 内置工具（Built-in Tools）
- 第三方成熟 Agent（如 Claude Code、Codex）
- 文件系统与沙盒

虚拟员工不直接在本地执行操作，而是通过**服务端转发指令**到已分配的工作环境节点执行，结果回传。

### 协作应用 (Collaboration App)

用户的交互入口。以类 Slack/飞书的形态呈现：

- **IM 通讯**：与虚拟员工的消息对话
- **协作工具**：文档、表格、审批流等工作产出载体
- **组织管理**：创建和管理组织、虚拟员工
- **上下文辅助**：为虚拟员工提供消息前置分析（标记、RAG），减少 token 消耗

### Agent 服务器 (Agent Server)

虚拟员工系统的服务端，由两部分组成：

| 组件 | 职责 |
|------|------|
| **接入层** | 对接协作应用，处理消息收发、协议转换 |
| **虚拟员工管理服务** | 管理虚拟员工生命周期、租户隔离、资源调度 |
