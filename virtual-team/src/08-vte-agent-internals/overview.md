# 虚拟员工 Agent 内部设计

## 概述

虚拟员工是面向用户的 Agent 实体，其内部不是单一 Agent，而是由**多个 VTA Agent 协作**构成的复合系统。

本章描述虚拟员工内部的 Agent 架构、工具体系和配置包规范。以下子章节展开：

| 子章节 | 内容 |
|--------|------|
| [内部 Agent 架构](./agent-architecture.md) | 意图 Agent / 主 Agent / 子 Agent 的职责、协作协议和交互流程 |
| [工具体系](./tool-system.md) | 远程工具与平台工具的双轨设计、工具路由、注册与发现 |

## 与 VTA 的关系

虚拟员工是 VTA 之上的**高阶封装**：

| 层面 | 关注点 | 核心概念 |
|------|--------|---------|
| **VTA Runtime** | Agent 推理循环、LLM 交互、工具调用 | Session、Turn、Message、Event、Approval |
| **虚拟员工层** | 消息处理、意图识别、工作上下文、多 Agent 协作 | 工作上下文、意图路由、Agent 编排、配置包 |

虚拟员工层不修改 VTA——它使用 VTA 提供的标准能力构建更复杂的 Agent 行为。

## 虚拟员工的状态管理

除了底层 VTA 的 Session/Message/Event 持久化外，虚拟员工层维护额外的状态：

- **工作上下文列表**：当前所有活跃/暂停/归档的工作上下文
- **消息关联映射**：协作应用消息 ID ↔ 工作上下文 ID
- **子 Agent 登记表**：当前活跃的子 Agent 及其状态
- **资源占用**：工作环境节点的分配和负载状态
- **在线状态**：虚拟员工在协作应用中的在线/离线/忙碌状态

### 状态持久化策略

| 状态类型 | 持久化方式 | 说明 |
|---------|-----------|------|
| 工作上下文 | Agent 服务器 Store | 与用户租户绑定，支持恢复 |
| 消息关联映射 | 协作应用 markers + Agent 服务器索引 | 双重存储保证一致性 |
| 子 Agent 登记 | Agent 服务器内存 + 定期快照 | 子 Agent 生命周期短，崩溃后可重建 |
| 资源占用 | Agent 服务器内存 | 重启后通过工作环境节点重新上报 |
| Session 级状态 | VTA Store（SQLite/PostgreSQL） | 由 VTA Runtime 独立管理 |
