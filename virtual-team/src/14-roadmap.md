# 路线图

## 当前阶段

Virtual Team 处于**架构设计与体系化文档阶段**。在进入具体实现前，先建立完整的项目级设计文档，确保从全局到细节的连贯性。

## 整体路径

Virtual Team 的开发路径遵循"**从整体到底层，从底向上实施**"的原则：

```
体系化设计文档 → VTA (Agent 基座) → Agent 服务器 → 虚拟员工封装层 → 协作应用 → 工作环境客户端
```

## 阶段规划

### 阶段 0：体系化设计文档（当前）

- 建立从项目愿景到技术细节的完整设计文档
- 明确两大子系统及其边界
- 定义核心概念和术语

此阶段的产出即为本文档。

### 阶段 1：VTA Agent Runtime 基座

基于已完成的架构设计，实现 Pure Agent 骨架：

- 完整 Agent 推理循环
- MessageStore 与双轨持久化
- Prompt 配置包管理
- 多 Provider 支持
- Sub-agent 能力

当前 VTA 的详细阶段划分见 `virtual-teams-agent/frozen-plan/`。

### 阶段 2：Agent 服务器

在 VTA 成熟后，构建 Agent 服务器：

- 接入层（协作应用协议适配）
- 虚拟员工管理服务（生命周期、多租户路由）
- 多租户数据隔离
- 虚拟员工实例的冷热管理

### 阶段 3：虚拟员工封装层

在 Agent 服务器之上构建虚拟员工的高阶封装：

- 多 Agent 协作（意图 Agent + 主 Agent）
- 工作上下文管理
- 消息处理机制
- 配置包的高阶扩展

### 阶段 4：协作应用核心

构建协作应用的第一个可用版本：

- IM 通讯核心
- 虚拟员工接入
- 组织管理
- 消息标记与上下文增强

### 阶段 5：工作环境客户端

构建工作环境客户端：

- 节点注册与心跳
- 工具执行与沙盒
- MCP 集成
- 文件系统隔离

### 阶段 6：协作应用完整版

协作应用的完整功能：

- 协作工具集（文档、表格、看板、审批流）
- 第三方 IM 渠道接入
- 移动端适配

### 阶段 7：商业化就绪

- 高级助理
- 云端工作环境
- 计费系统
- SLA 监控

### 阶段 8：开放平台

- 第三方虚拟员工配置包市场
- 开发者工具与文档
- 应用审核与发布流程

## 里程碑

| 里程碑 | 内容 | 可交付物 |
|--------|------|---------|
| M1 | VTA 最小可运行 Agent | 本地 LLM + tool call + MCP |
| M2 | VTA 完整 Agent Runtime | Prompt 配置包 + 双轨持久化 + Sub-agent |
| M3 | Agent 服务器可用 | 虚拟员工创建/管理/消息路由 |
| M4 | 内测版 | 协作应用 + 虚拟员工 + 基本工作环境 |
| M5 | 公测版 | 高级助理 + 完整协作工具 |
| M6 | 商业化 | 云端工作环境 + 计费 + 开放平台 |
