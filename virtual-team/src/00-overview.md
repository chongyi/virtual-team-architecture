# 项目总览

## Virtual Team 是什么

Virtual Team（虚拟团队）是一个面向**无技术背景用户**的 AI Agent 应用平台。它以类 Slack/飞书的协作应用形态作为用户入口，让用户像管理真实团队一样创建、配置和使用 AI 虚拟员工。

## 为什么需要 Virtual Team

当前 AI Agent 生态虽然丰富，但大量产品面向的是具备技术背景的用户：

- 命令行安装和交互是主流形态
- 概念繁多（Skills、MCP、Prompt、Token、Context Window 等）学习成本高
- 权限和安全配置依赖用户自身判断
- 不同 Agent/模型的维护和版本管理增加负担
- LLM 在注意力机制、长上下文、细节保留等底层限制进一步放大使用门槛

Virtual Team 的目标是让 AI Agent **回归应用本身**——用户不需要理解 Agent 如何工作，只需要知道"我雇了一个能帮我做事的人"。

## 两大子系统

Virtual Team 由两个核心子系统构成：

| 子系统 | 职责 | 类比 |
|--------|------|------|
| **协作应用** | 用户入口，提供 IM 通讯、协作工具、组织管理 | Slack / 飞书 |
| **虚拟员工系统** | Agent 运行时，管理虚拟员工、处理消息、执行工作 | 人力资源公司 + 员工大脑 |

两者通过协议层对接。协作应用本身是一个完整的即时通讯协作工具，虚拟员工通过专门的协议接入其中，作为"联系人"出现在用户面前——就像真人同事一样。

## 核心理念

### 现实拟合

Virtual Team 的设计不是凭空创造新概念，而是把现实世界中**公司组织运营**的成熟模型映射到 AI Agent 系统中：

- 用户 → 公司拥有者/创始人
- 组织 → 公司及其部门
- 虚拟员工 → 真实员工（有岗位、能力、工具）
- 助理 → 管理型员工（协调、分配、汇报）
- 工作区 → 员工的独立工作台

### 零门槛

用户首次进入应用，默认提供一个私人小助理。这个助理引导用户完成一切——建立组织、招募虚拟员工、分发任务。用户不需要阅读文档、配置参数或理解 AI 概念。

### Pure Agent 骨架

虚拟员工底层的 Agent Runtime（VTA）遵循"空性"原则：零预设 prompt、零内置工具、零默认技能。所有能力通过配置包注入，保证每个虚拟员工的纯粹性和独特性。
