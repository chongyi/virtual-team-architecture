# 商业化模型

## 概述

Virtual Team 的商业化策略与产品定位一致——**面向非技术用户，零门槛使用**。

## 目标用户

### 核心用户画像

| 维度 | 描述 |
|------|------|
| **技术背景** | 无或极低——不要求编程、命令行或 AI 概念知识 |
| **工作场景** | 知识工作者、中小企业主、自由职业者、团队管理者 |
| **核心痛点** | 有大量重复性脑力工作，但无法利用现有 AI 工具（门槛太高） |
| **付费意愿** | 中高——如果产品确实能节省时间/替代人力 |
| **决策模式** | 类似 SaaS 工具采购——看 ROI 而非技术指标 |

### 典型使用场景

| 场景 | 用户类型 | 虚拟员工需求 |
|------|---------|-------------|
| 电商运营 | 淘宝/Shopify 店主 | 客服、数据分析、商品描述撰写、竞品监测 |
| 内容创作 | 自媒体/博主 | 文案撰写、素材整理、排期管理、数据分析 |
| 小型企业管理 | 中小企业主 | 财务数据整理、HR 流程、销售跟踪、报告生成 |
| 自由职业 | 设计师/咨询师 | 项目管理、客户沟通记录、发票和合同管理 |

## 商业化载体

### 助理（Assistant）

用户首次进入应用时免费获得一个**基本助理**。助理是用户与虚拟团队交互的第一个触点，负责引导用户完成初始化：建立组织、招募虚拟员工、学习使用方式。

**高级助理**（付费）通过配置包提供更强的能力：

| 能力 | 基本助理（免费） | 高级助理（付费） |
|------|----------------|-----------------|
| 底层模型 | 低成本模型（Haiku/GPT-4o-mini） | 高级模型（Sonnet/Opus/GPT-4.1） |
| 组织管理 | 最多 3 个组织 | 无限制 + 智能组织建议 |
| 虚拟员工数量 | 最多 3 个 | 无限制 |
| 任务分派 | 人工指定 | AI 自动分析 + 人工确认 |
| 主动管理 | 被动响应 | 主动提醒、定期汇总汇报 |
| 工作上下文 | 3 个并发 | 10 个并发 |

高级助理的能力差异完全在虚拟员工框架内实现——通过选择更好的模型、更多的工具、更高级的管理型 prompt——而非底层架构的特殊分支。

### 云端工作环境

为不希望自行维护本地环境的用户提供托管工作环境：

- 类似云主机/云 PC 的订阅模式
- 预装工作环境客户端，优化定制
- 内置常用工具（浏览器、文件系统、MCP Server）
- 按资源配置和实际使用时长计费
- 企业版提供专属工具（数据库连接、内网 API）

### 第三方配置包市场

远期开放后，第三方开发者可提交虚拟员工配置包到市场：

- 用户浏览和购买第三方配置包
- 平台与开发者分成（参考 App Store 30% 模式，或更低的 15-20% 以激励开发者）
- 用户评价和评分机制保障质量
- 平台提供沙盒测试环境，安装前可试用

## 竞争壁垒

### 用户使用习惯沉淀

用户使用习惯的沉淀——组织架构、虚拟员工配置、工作上下文历史——形成自然的数据迁移成本。用户不是被"锁定"而是"不值得迁移"：

- 迁移意味着重建组织架构、重新训练/配置虚拟员工、丢失历史工作上下文
- 随着使用深入，沉淀成本持续增加
- 但始终提供数据导出能力（不搞 lock-in 的恶意锁定）

### 虚拟员工生态

第三方开发者可开发虚拟员工配置包或工作环境工具，形成双边市场效应。

```
用户越多 → 市场越大 → 开发者越有动力
    ↑                        ↓
用户体验越好 ← 更多优质虚拟员工可用
```

### 极致易用性

核心壁垒不是技术而是体验——无技术背景用户能在 5 分钟内开始使用虚拟员工完成实际工作。这是一条需要持续打磨的护城河。

当前市场上不存在"零门槛 AI Agent 平台"——所有竞品要么面向开发者（Claude Code、Codex），要么面向特定垂直场景（客服 Bot），要么需要理解 AI 概念。

### 平台效应

远期开放平台后，形成"用户 - 虚拟员工 - 开发者"的三边网络：

- 用户获得丰富的虚拟员工选择
- 开发者获得收入
- 平台获得网络效应和数据飞轮

## 市场定位

### 竞争格局

| 产品类别 | 代表产品 | 与 Virtual Team 的关系 |
|---------|---------|----------------------|
| **AI 编码 Agent** | Claude Code, Codex, Cursor | 垂直领域（编码），互为补充——可通过工作环境节点接入 |
| **通用 AI 助手** | ChatGPT, Claude.ai | 对话式交互，无组织/多 Agent 概念 |
| **无代码 AI** | Zapier AI, Make | 工作流自动化，缺少对话式交互和多 Agent 协作 |
| **企业 IM** | Slack, 飞书 | Virtual Team 的协作应用对标它们，但内置了 Agent 系统 |
| **客服 Bot** | Intercom, Zendesk AI | 单一场景（客服），Virtual Team 是多场景平台 |

Virtual Team 的独特定位：**以协作应用为入口、以多 Agent 虚拟团队为核心的通用 AI 工作平台**。目前无直接竞品。

### 进入市场的路径

1. **第一阶段（内测）**：邀请制，100-500 种子用户，验证核心体验
2. **第二阶段（公测）**：开放注册，基础功能免费，收集产品数据
3. **第三阶段（商业化）**：推出付费层，优化转化漏斗
4. **第四阶段（平台化）**：开放配置包市场，引入第三方开发者
