# 商业化模型

## 概述

Virtual Team 的商业化策略与产品定位一致——**面向非技术用户，零门槛使用**。

## 商业化载体

### 助理（Assistant）

用户首次进入应用时免费获得一个**基本助理**。助理是用户与虚拟团队交互的第一个触点，负责引导用户完成初始化。

**高级助理**（付费）提供更强大的能力：

- 更好的底层模型（更强的分析、协调和决策能力）
- 更复杂的管理能力（大规模组织、多业务线协调）
- 更多的任务分派策略和优化建议
- 主动式管理（无需用户每步操作，自主协调虚拟员工完成目标）

高级助理的能力差异完全在**虚拟员工框架内**实现——通过配置包选择更好的模型、更多的工具、更高级的管理型 prompt，而非底层架构的特殊分支。

### 云端工作环境

为不希望自行维护本地环境的用户提供托管工作环境：

- 类似云主机/云 PC 的订阅模式
- 预装工作环境客户端，优化定制
- 内置常用工具和第三方 Agent
- 按资源配置和使用时长计费
- 可选的专属工具（企业级数据库连接、专用 API 等）

## 竞争壁垒

### 用户锁定

用户使用习惯的沉淀——组织架构、虚拟员工配置、工作上下文历史——形成自然的数据迁移成本。用户不是被"锁定"，而是"不值得迁移"。

### 虚拟员工生态

第三方开发者可开发虚拟员工配置包或工作环境工具，形成双边市场效应。用户越多 → 开发者越有动力 → 可用虚拟员工越多 → 用户越多。

### 极致易用性

核心壁垒不是技术而是体验——无技术背景用户能在 5 分钟内开始使用虚拟员工完成实际工作。这是一条需要持续打磨的护城河，不是单点技术突破能替代的。

### 平台效应

远期开放平台后，形成"用户 - 虚拟员工 - 开发者"的三边网络，类似应用商店的生态壁垒。

## 收入来源

| 层级 | 产品 | 说明 |
|------|------|------|
| 免费 | 基础版 | 1 个基本助理 + 有限虚拟员工数 + 基础工具 |
| 付费 - 助理 | 高级助理 | 更强模型、更复杂管理、更大规模组织 |
| 付费 - 环境 | 云端工作环境 | 托管执行环境、按需订阅 |
| 付费 - 企业 | 企业版 | 专属部署、SLA 保障、定制开发、管理后台 |

## 远期的开放平台

- 开发者可提交虚拟员工配置包到市场
- 用户可浏览和安装第三方虚拟员工
- 类似应用商店的分成模型
- 支持私有化部署（面向大型企业客户）
